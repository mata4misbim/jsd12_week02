MongoDB  เหมือน โฟลเดอร์เก็บใบเสร็จ แบบที่พี่ในแคมป์เคยอธิบายไว้ จำไม่ได้ว่าคนไหน น่าจะพี่ตรี

เก็บข้อมูลเป็นแผ่นๆ เรียกว่า BSON แล้วก็ JSON

แล้วก็ตรงนี้ไม่ได้กำหนดว่าทุกอย่างต้องเหมือนกันแต่ละแผ่นหน้าตาไม่เหมือนกันก็ได้ แผ่นแรกอาจมี 3 หัวข้อของข้อมูล แผ่นสองมี 10 หัวข้อ

เรื่องพวก Syntax ที่จำได้ก็มีประมาณนี้ find() = การค้นหา , insertOne() = เพิ่มข้อมูล , updateOne() = แก้ไขข้อมูลหรืออัพเดทข้อมูลให้เป็นล่าสุด การใช้ Syntax จะคล้ายๆ 
การหาข้อมูลใน Object ของ JavaScript น่าจะเพราะโครงการเหมือนกัน

Syntax ของ MongoDB จะทำงานเหมือน Object ใน JavaScript โดย find() ใช้ค้นหาข้อมูล, insertOne() ใช้เพิ่มข้อมูลใหม่ 1 รายการ และ updateOne() ใช้แก้ไขข้อมูลที่มีอยู่เดิมโดยระบุเงื่อนไขการหาและใช้คำสั่ง $set เพื่อเปลี่ยนเฉพาะจุดที่ต้องการได้

# วิธีจำ Syntax = ในรูปแบบการดึงข้อมูล MongoDB = แปลเหมือนเรียกใช้ฟังก์ชัน JavaScript

db.users.find({ id: 1 })

# แปล "เข้าถึงฐานข้อมูล (db) -> ไปที่คอลเลกชัน users -> แล้ว หา (find) คนที่มี id เป็น 1"

# รูปแบบเหมือน Object ใน JavaScript

# การเพิ่มข้อมูลในตาราง
db.users.insertOne({ name: 'Oh', age: 21 })

# แปลโค้ดข้างบนคือ "ในฐานข้อมูลนี้ (db) ไปที่ users แล้ว ใส่ข้อมูลหนึ่งตัว (insertOne) คือ ชื่อ Oh และ อายุ 21" (เป็นการส่ง Object {} เข้าไปเหมือน JavaScript)

# การแก้ไขข้อมูล หรืออัพเดท
ตรงนี้เหมือนกัน ยกตัวอย่างการเปลี่ยนชื่อ

db.users.updateOne({ id: 1 }, { $set: { name: 'Metha' } })

# แปล " ในฐานข้อมูลนี้ (db) ไปที่ users แล้ว อัปเดตหนึ่งตัว (updateOne) ค้นหาคนที่มี id เป็น 1 แล้วทำการ ตั้งค่า ($set) ให้ ชื่อใหม่คือ 'Metha' "

# การลบข้อมูล 
Ex. เหมือนกัน คือลบข้อมูล คนที่มรไอดี = 1

db.users.deleteOne({ id: 1 })

# แปล "ในฐานข้อมูลนี้ (db) ไปที่ users แล้ว ลบทิ้งหนึ่งตัว (deleteOne) คือคนที่มี id เป็น 1"
